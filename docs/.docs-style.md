# DuckChat 文档样式指南

本文档定义了 DuckChat 项目文档的统一格式和样式规范。

## 📋 文档结构规范

### 1. 文档头部格式
```markdown
# 文档标题

简短的文档描述，说明本文档的目的和内容范围。

## 目录（可选，长文档使用）
- [章节1](#章节1)
- [章节2](#章节2)
```

### 2. 章节标题层级
```markdown
# 一级标题 - 文档标题
## 二级标题 - 主要章节
### 三级标题 - 子章节
#### 四级标题 - 具体内容
##### 五级标题 - 详细说明
###### 六级标题 - 极少使用
```

### 3. 表格格式
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

## 🎨 格式规范

### 1. 代码块
```cpp
// C++ 代码示例
#include <iostream>
int main() {
    return 0;
}
```

```bash
# Shell 命令示例
make all
./build/duckchat-server
```

```json
// JSON 示例
{
  "key": "value"
}
```

```sql
-- SQL 示例
SELECT * FROM users WHERE id = 1;
```

### 2. 强调和标记
- **粗体** - 重要概念和关键词
- *斜体* - 强调和术语
- `代码` - 变量名、文件名、命令
- ~~删除线~~ - 过时内容
- <u>下划线</u> - 需要注意的内容

### 3. 列表格式
#### 无序列表
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2

#### 有序列表
1. 第一步
2. 第二步
   1. 子步骤2.1
   2. 子步骤2.2

#### 任务列表
- [x] 已完成任务
- [ ] 待完成任务
- [-] 进行中任务

### 4. 链接格式
```markdown
[链接文本](./path/to/file.md)
[代码引用](../path/to/code.cpp:123)
[外部链接](https://example.com)
```

## 🎯 内容规范

### 1. 技术术语
- 首次出现的技术术语提供简要说明
- 保持术语使用的一致性
- 避免过度使用缩写，必要时提供全称

### 2. 代码示例
- 提供完整可运行的示例
- 包含必要的注释说明
- 标注示例的预期输出

### 3. 错误处理
- 明确指出常见的错误情况
- 提供具体的错误信息示例
- 给出解决方案和预防措施

## 📝 特殊元素


## 🔧 工具和检查

### 1. Markdown 检查
- 使用 Markdownlint 检查格式规范
- 确保所有链接有效
- 检查图片和资源引用

### 2. 拼写检查
- 使用拼写检查工具
- 统一技术术语的拼写
- 注意中英文混排的空格

### 3. 格式化工具
```bash
# 使用 prettier 格式化 Markdown
npx prettier --write docs/*.md

# 使用 markdownlint 检查
npx markdownlint docs/*.md
```

## 📚 最佳实践

### 1. 文档组织
- 按逻辑顺序组织内容
- 使用目录导航长文档
- 保持章节长度适中

### 2. 内容更新
- 代码变更时同步更新文档
- 定期审查文档的准确性
- 标注文档的最后更新时间

### 3. 用户体验
- 提供清晰的导航结构
- 使用一致的格式和样式
- 确保文档的可搜索性
